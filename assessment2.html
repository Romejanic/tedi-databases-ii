Databases II Final Assessment

Create a table in a database. This database must include at least 10 fields, with at least 50 records. You can represent this table, simply, in an excel spreadsheet. Throw entity integrity and referential integrity out of the window, as well as normalisation. Try to create something that will break.

For example: Create a party guest list. Include first name, surname, age, suburb, etc.
The key in this database is the guests first name.

Then, give this database to a partner, and recieve their database. Fix your partners database.
